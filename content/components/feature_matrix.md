# Feature Matrix / Comparison Table

**Definition**: A dense, grid-based layout used to explicitly compare the granular features of different products, plans, or competitors side-by-side. It consists of a vertical list of features (the rows) intersecting with different product variations (the columns), usually filled with simple checkmarks (`✓`) or crosses (`✕`) to indicate inclusion, or short text strings for specific limits.

**History**: Inherited directly from classic consumer reporting magazines (like *Consumer Reports*) comparing physical appliances. On the web, as Software as a Service (SaaS) products became increasingly complex, standard "Pricing Tables" ran out of room. Marketing teams couldn't fit 50 features into a tiny pricing "Card." The solution was to decouple the high-level price from the granular details, placing a massive, scrolling Feature Matrix immediately below the Pricing Table.

**Variations**:
- **The Classic Checkmark Matrix**: The left column lists every feature. The subsequent columns represent Plan tiers. A simple green check indicates the feature is included.
- **Competitor Comparison**: A hyper-aggressive but highly effective marketing tool. The left column lists features. The center column is "Our Product" (showing all green checks). The right column is "Our Biggest Competitor" (showing red X's for missing features).
- **Categorized / Expandable Matrix**: When a product has literally 100 features, the rows are grouped under bold category headers (e.g., "Security", "Analytics", "Support"). Sometimes these categories function as accordions, allowing the user to hide rows they don't care about.
- **Sticky Column Header Matrix**: Essential for incredibly long tables. As the user scrolls down to read the features, the Column Headers (e.g., "Free", "Pro", "Enterprise") stick to the top of the viewport so the user never forgets which tier they are looking at.

**Use Cases**:
- **Enterprise SaaS Purchasing**: IT managers buying software absolutely require the exhaustive matrix to ensure compliance and technical capability before signing a massive contract.
- **Hardware Spec Sheets**: Buying a laptop or a phone. Users need a clean grid to compare "RAM," "Processor," and "Battery Life" between the Base and Pro models.
- **Hosting / Infrastructure**: Comparing the exact specifications of different cloud server configurations.
